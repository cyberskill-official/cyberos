## §1 — Statement (BCP-14 normative)

1. The SKILL OCI registry **MUST** serve manifest reads (`GET /v2/<name>/manifests/<ref>`) and blob reads (`GET /v2/<name>/blobs/<digest>`) at **≥ 99.95% monthly availability** measured at the public ingress.
2. Read availability is the primary SLO; **write availability (publish path) is permitted to be lower (99.5%)** — temporary publish outages are acceptable, runtime pull outages are not (they break running skills).
3. Bundle blob delivery **MUST** be served from a CDN edge or equivalent caching layer; first-byte latency p95 < 500ms globally.
4. Manifest reads **MUST** be served with `Cache-Control: max-age=300` plus an immutable digest header so the runtime can safely re-pull.
5. Registry downtime > 5 minutes triggers a sev-2 alert; downtime > 30 min triggers sev-1.

## §2 — Why this constraint

Every CUO workflow invocation pulls the skill bundle on cold start. If the registry goes down, the entire platform's skill execution path goes down — even though every other service is healthy. 99.95% (≈ 21.6 minutes/month) is the realistic ceiling for a single-region OCI registry behind a CDN; if we need 99.99%, we'd dual-region the registry. The asymmetric write SLO (99.5%) is the trade-off: publishing is a low-frequency operator action, while pulling is on the hot path. The CDN+max-age combination amortises pull cost — most pulls never touch origin.

## §3 — Measurement

- Synthetic prober every 60s from 3 regions hitting `GET /v2/cyberos-skill-fr-author/manifests/latest`.
- Availability calculation: (successful prober pings / total prober pings) × 100, monthly window.
- Histogram `skill_registry_pull_latency_seconds{region, op=manifest|blob}`.

## §4 — Verification

- Monthly availability report generated by OBS — flag any month < 99.95%.
- Chaos drill (A, quarterly) — registry simulated outage; assert CUO workflows fall back to in-memory bundle cache (FR-SKILL-101) and continue serving for at least 15 minutes.
- Synthetic prober uptime ≥ 99.99% (instrumentation reliability).

## §5 — Failure handling

- Registry 5xx burst > 1% for 5 min → sev-2; on-call investigates origin + CDN cache state.
- Total registry down > 5 min → sev-1; activate read-only failover bundle cache; pause publishes.
- Monthly SLO miss → quarterly postmortem; CTO decides whether to dual-region.
