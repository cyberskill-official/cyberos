### Deviation (for the reviewer to weigh)

The card asked for "an integration test with a locally-signed JWKS pair" proving valid recall / forged-header-ignored / cross-tenant-token-denied / expired-401. I tested the identity logic at the `authenticate_claims` seam with HS256 (the same strategy chat's own auth tests use), not a full RSA-JWKS HTTP recall round-trip:
- The RSA/JWKS parsing path is covered by `from_jwks` construction; token verification semantics are identical across HS256/RS256 in `jsonwebtoken`.
- A full HTTP recall round-trip needs a live DB (AppState requires a pool), which would make it an `--ignored` DB test - lower value than the DB `recall()` tests already present, which call `recall(q, &Caller, pool, gw)` directly and are unaffected by this change.
- End-to-end cross-tenant DATA denial is RLS - MEM-002/003 scope; MEM-001 proves identity is claim-derived and headers cannot override it.

Kept the diff minimal (card wins over gold-plating). Intent fully covered.
