## MEM-002 - Fail-closed RLS on brain tables (R74, F16) -> in_review

Migrations 0006/0007/0008 shipped each brain-table policy with three arms: the real-tenant arm, a nil-uuid "admin bypass", and an `IS NULL` (unset-GUC) arm that FAILS OPEN. A code path that forgets `brain::tenant_tx` therefore read every tenant's rows instead of none.

Change:
- `services/memory/migrations/0009_rls_fail_closed.sql` (new) - recreates all 4 brain-table policies (`brain_event_embedding`, `brain_summary`, `brain_ingest_cursor`, `brain_tier_watermark`) with ONLY `tenant_id::text = current_setting('app.tenant_id', true)` for USING + WITH CHECK. ENABLE + FORCE re-asserted. An unset GUC now yields `tenant_id = NULL` -> no rows (fail-closed).
- `services/memory/src/brain/mod.rs` - removed the unused `pub const ADMIN_TENANT` (nil-uuid) and documented the fail-closed contract on `rls_set_tenant`.
- `services/memory/tests/brain_rls_test.rs` (new, `#[ignore]`) - proves the guarantee at the DB.

### Why a probe role (important)

The dev `cyberos` user is a Postgres SUPERUSER, and superusers bypass RLS entirely - so the existing brain "RLS" tests (which connect as `cyberos`) never actually exercised RLS. `brain_rls_test` creates a throwaway `NOSUPERUSER NOBYPASSRLS` role, `SET ROLE`s to it, and asserts:
1. no `app.tenant_id` set -> `count(*) = 0` on all 4 tables (the F16 hole, now closed);
2. `app.tenant_id = <other tenant>` -> 0 (cross-tenant invisible);
3. `app.tenant_id = <owner>` -> row visible (the normal tenant_tx path still works).

### Deviation - no dedicated admin role (card asked for one)

The card's step "create cyberos_memory_admin BYPASSRLS role for rebuild/backfill" was NOT implemented, by design: I verified every brain-table access is per-tenant via `tenant_tx` (the ingest/tiering/summary workers, the recall handler, `cyberos-memory-admin rebuild|brain-*` which all require `--tenant <UUID>`, and `brain::backfill` which uses `tenant_tx` throughout). No path relied on the nil-uuid bypass, so removing it needs no replacement role. The genuine cross-tenant readers are `discover_tenants` and the `/metrics` endpoint, which read `l1_audit_log` / `l2_ingest_cursor` (NOT brain tables) - those get RLS + the dedicated admin role in MEM-003. Recorded so MEM-003 picks it up. Kept the MEM-002 diff minimal (card intent = fail-closed; achieved).
