## MEM-005 - Fix the dead recall confidence floor (R9, F7) -> in_review

`recall::recall` set `best_summary = 1.0` whenever ANY summary matched (`summary_hits.first().map(|_| 1.0)`), so the `recall_confidence_floor` (default 0.30) -> drill decision could never fire on quality grounds â€” a below-floor summary would never trigger a drill into raw hot events.

Change (`src/brain/recall.rs`):
- `summary_search` now returns `(Vec<Candidate>, f32)` where the `f32` is the REAL top summary similarity: the vector SELECT adds `(embedding <=> $2::vector) AS cos_dist`, and `best = 1 - cos_dist` of the closest row (rows are ordered by ascending cosine distance). The full-text fallback (no vector) reports `0.0` â€” drill needs a vector regardless.
- `recall` uses that value and decides drill via a new pure helper `should_drill(explicit, best_summary, floor)` instead of the hardcoded `1.0`.
- `best_summary` is now surfaced in the `explain` envelope (also serves R23).

Tests:
- Unit: `drill_fires_when_best_summary_below_floor` (weak -> drill; strong -> no drill; explicit always drills; and best_summary=1.0 would NOT drill â€” pinning that the value must be a real similarity).
- DB regression (`tests/brain_confidence_floor_test.rs`, `#[ignore]`): seed 6 events -> ingest + summarise -> recall with a query dissimilar to the verb-frequency digest, `drill=false`, `explain=true`; assert `explain.best_summary` is in `[0,1)`. The old hardcoded `1.0` fails this; the fix passes it.
