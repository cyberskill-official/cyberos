### Accepted with rationale (Dependabot dismissed)

- opentelemetry_sdk (GHSA-w9wp-h8wv-79jx, medium) - alert #47 dismissed as `not_used`. The advisory is unbounded memory allocation in W3C Baggage propagation. CyberOS uses only W3C TraceContext propagation (TraceContextPropagator, services/ai-gateway/src/otel/propagation.rs); the Baggage propagator is never instantiated and no baggage header is parsed anywhere in services, so the vulnerable path is unreachable. A 0.27 -> 0.32 major migration of the otel stack (deployed ai-gateway + obs-collector + the shared obs- sdk RED/exemplar/tracecontext code, ~10 files, no way to runtime-test the telemetry here) was judged disproportionate for an unreachable medium advisory. Revisit + bump the otel stack to >= 0.32.1 if Baggage propagation is ever adopted.
- glib (medium) - alert #35 dismissed as `tolerable_risk`. glib 0.20 is unreachable: tauri 2.11 pins the gtk-rs 0.18 stack (gtk/gdk/glib 0.18) via wry/webkit2gtk, and glib 0.20 needs the gtk-rs 0.20 ecosystem that tauri 2.x does not target yet. apps/desktop is a local operator desktop shell (not a deployed network service) and glib is a Linux GUI transitive. Re-address on the next tauri/wry release that moves to gtk-rs 0.20.
