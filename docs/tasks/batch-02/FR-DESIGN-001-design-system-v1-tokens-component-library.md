# Feature Request

> Turn Your Will Into Real.

## Summary

Ship the **CyberSkill Global Design System v1.0.0** (locked 2026-04-25 per PRD §8.9) as the canonical visual substrate for every CyberOS Module-Federation remote. Deliver the `@cyberskill/tokens` npm package exporting design tokens in W3C DTCG format (2025.10), the React component library at parity with Design System parts 3a–3h, the typography stack (Be Vietnam Pro UI for Vietnamese-aware diacritic-correct rendering; JetBrains Mono for code), the colour anchors (Umber #45210E, Ochre #F4BA17, sub-brand accents), spacing/elevation/motion tokens, accessible-contrast pairings, dark/light surface mappings, and the **`genie-tokens`** sub-package that governs the Genie panel sidebar, confidence/citation chips, mode-indicator (Notify/Question/Review) colours, and persona-version footer typography. Every P0 module's "Module Ready" gate (PRD §7.2) fails without consuming the tokens package.

## Problem

A 22-module platform without a design system produces 22 visual languages. The PRD §8.9 commits to one published source for tokens precisely because the moment two modules diverge on spacing rhythm or typography, the host shell ceases to feel like one product. Three failure modes a small team must avoid:

- **Hardcoded colours and spacings drifting per module.** Without a token package, every module owner re-types `padding: 16px` and `color: #45210E`; six months later the team has six "umbers".
- **Vietnamese typographic regressions.** Be Vietnam Pro is the chosen typeface specifically for diacritic-correct rendering of Vietnamese; system-default fonts mishandle stacked-diacritic glyphs (Ố, Ữ). A module that imports a different font silently breaks Vietnamese readability.
- **Genie panel inconsistency.** The Genie panel is a cross-module surface owned by the GENIE module but rendered inside every other module's pages. Without a dedicated `genie-tokens` sub-package, a CHAT-module developer can accidentally reskin the panel.

S0-1 sprint exit (PRD §17.1): "Design system 1.0 implemented in the host shell; component library at parity with parts 3a–3h of the design system documentation." The host shell already consumes the tokens (FR-INFRA-001 §"Design tokens"); this FR ships the **package itself, the components, the docs, and the lint enforcement**.

## Proposed Solution

The shape of the answer is a single npm workspace `cyberskill/design-system` with two published packages (`@cyberskill/tokens`, `@cyberskill/components`), a Storybook-based docs site, a lint enforcement rule, and a Figma sync.

**Token package — `@cyberskill/tokens`.**

Tokens are authored in W3C DTCG JSON format (2025.10 update) and built with Style Dictionary into the consumption formats:

- CSS custom properties (`@cyberskill/tokens/css/tokens.css`).
- Tailwind plugin (`@cyberskill/tokens/tailwind/plugin.js`) extending the Tailwind config.
- TypeScript type definitions (`@cyberskill/tokens/types/index.d.ts`) for editor autocompletion.
- Style Dictionary JSON (preserved for downstream consumers).
- iOS-format `.swift` (P3 mobile).
- Android-format `.xml` (P3 mobile).

Token categories (PRD §8.9):

- **Colour anchors.** Umber `#45210E` (primary), Ochre `#F4BA17` (accent), Cream `#FBF6E9` (page background, light), Charcoal `#1A1612` (page background, dark), and sub-brand accents (Lapis `#1F4068`, Jade `#2C7A4B`, Vermillion `#B5462E`, Slate `#5C6973`). Each anchor has a 11-step ramp (50–950) generated by `chroma-js` ensuring AAA contrast pairings between adjacent steps.
- **Surface mappings.** Light + dark themes with consistent semantic names (`surface.canvas`, `surface.elevated`, `surface.popover`, `surface.warning`, etc.). The host shell switches themes via CSS class on `<html>` so all consumers re-theme together.
- **Typography.** Be Vietnam Pro for `font.body`, `font.heading`, `font.label`, `font.caption`. JetBrains Mono for `font.code`, `font.kbd`. The fonts ship as self-hosted woff2 in the `@cyberskill/tokens/fonts/` subpath (no Google Fonts CDN — residency-clean).
- **Type scale.** Modular scale 1.250 (major third). 13 steps from `text.xs` (12px) to `text.6xl` (60px). Line-heights are paired (`leading.body` = 1.55 for body; `leading.heading` = 1.15 for headings; `leading.code` = 1.5 for code).
- **Spacing rhythm.** Eight-point grid: `space.0 .. space.32` mapping to `0px .. 128px` in 8-point increments + half-steps `space.0_5`, `space.1_5`, `space.2_5` for the rare 4-px adjustment.
- **Elevation.** Six tiers `elevation.0 .. elevation.5` rendering as `box-shadow` stacks with theme-aware tints (light tints in dark mode; charcoal tints in light mode).
- **Motion.** Durations `duration.instant` (50ms) → `duration.slow` (800ms); easing curves `easing.standard`, `easing.emphasised`, `easing.decelerated`, `easing.accelerated` (matching Material Design 3 semantics adapted to CyberOS rhythm).
- **Border radii.** `radius.0` (0px) through `radius.full` (9999px) with intermediate `radius.xs` (2px), `sm` (4px), `md` (8px), `lg` (12px), `xl` (16px), `2xl` (24px).
- **Z-index registry.** Named layers (`z.base`, `z.dropdown`, `z.sticky`, `z.popover`, `z.modal`, `z.toast`) so no module guesses a magic number.

**Genie tokens — `@cyberskill/tokens/genie`.**

A versioned sub-package governing every Genie-panel surface:

- Sidebar width (default 380px; collapsed 64px; expanded 520px).
- Mode-indicator colours: Notify (Ochre), Question (Lapis), Review (Vermillion), Paused (neutral 600).
- Confidence chip variants: low (warning), medium (info), high (success).
- Citation chip styling.
- Persona-version footer typography (`text.xs` JetBrains Mono).
- Mascot-state colour overlays (idle, thinking, answering, error, deferring, paused).

The genie-tokens version is bumped *together with* the persona version (FR-GENIE-001 §"Dual-sign persona versioning"); when the founder + Engineering Lead sign a new persona version, the matching genie-tokens version ships in the same release.

**Component library — `@cyberskill/components`.**

A React 19 component library covering:

- Layout: `<Stack>`, `<Inline>`, `<Grid>`, `<Container>`, `<Divider>`, `<Spacer>`.
- Typography: `<Heading>`, `<Text>`, `<Code>`, `<Kbd>`, `<Citation>`.
- Form: `<Button>`, `<TextField>`, `<TextArea>`, `<Select>`, `<Combobox>`, `<Switch>`, `<Checkbox>`, `<RadioGroup>`, `<Slider>`, `<DatePicker>`, `<DateRangePicker>`, `<TimezoneSelect>`, `<FileUpload>`.
- Data display: `<Card>`, `<Badge>`, `<Chip>`, `<Tag>`, `<Avatar>`, `<AvatarStack>`, `<Tooltip>`, `<Popover>`, `<Modal>`, `<Drawer>`, `<Toast>`, `<EmptyState>`, `<Skeleton>`.
- Navigation: `<Tabs>`, `<Breadcrumbs>`, `<Pagination>`, `<Sidebar>`, `<NavigationItem>`.
- Feedback: `<Alert>`, `<Spinner>`, `<ProgressBar>`, `<Stepper>`.
- Domain primitives: `<MemberAvatar>`, `<MemberPicker>`, `<TenantBadge>`, `<ConfidenceChip>`, `<CitationChip>`, `<PersonaVersionFooter>`, `<DiffView>` (used by FR-BRAIN-CONFLICT-001 + FR-BRAIN-NLCRUD-001).

Every component:

- Forwards refs.
- Has a TypeScript-inferred prop API.
- Has Storybook stories with all variants and dark/light themes.
- Has axe-core a11y tests in CI; AAA contrast on all default token pairings.
- Renders in vi-VN and en-US locale samples in Storybook (catches mistaken English-only copy in the component itself).
- Is tree-shakeable; per-import bundle size is reported and fails CI if a single component import exceeds 8 KB gzipped.

**Storybook docs site.**

`design.cyberskill.world` (private to start; public at P4 once external customers exist). Hosts:

- Token reference (every token + its computed value + a swatch / preview).
- Component catalogue with live examples.
- Pattern guidelines (when to use `<Modal>` vs `<Drawer>`; the persona-confidence-chip rules).
- Vietnamese-typography do/don't gallery.
- Accessibility checklist per component.
- Migration guide from one major design-system version to the next.

**Lint enforcement.**

A small ESLint plugin `@cyberskill/eslint-plugin-tokens-only` is published. Every module's `eslint.config.js` extends the plugin. The plugin rejects:

- Hardcoded hex colours in TSX/JSX/CSS files (must reference a token).
- Hardcoded pixel spacings outside the 8-point rhythm.
- Imports of non-allowlisted fonts.
- Inline `style={{}}` props for properties governed by tokens.
- Direct DOM manipulation that bypasses the component library for governed primitives (toasts, modals, tooltips).

Rule violations are blocking errors; CI fails the PR. Module owners can request a token addition rather than disabling the rule.

**Figma sync.**

Tokens are exported from the Style Dictionary build to a Figma plugin that updates the Figma library on every release. The Figma library is the canonical design surface; engineers should never need to ask "what colour goes here" — they read the token name from the Figma layer.

**Versioning & release.**

`@cyberskill/tokens` and `@cyberskill/components` follow semver. The genie-tokens sub-package follows the persona-version semver (linked in `@cyberskill/tokens/package.json`'s `peerDependencies`). Major-version bumps require a migration note + a one-quarter deprecation window. PRs to the design-system repo are reviewed by the founder + (when hired) the Designer; in P0, the founder + a contracted designer (one of the 10 employees with design background) act as approvers.

**MCP tools.** Read-only.

- `cyberos.design.list_tokens(category?)` — returns tokens for runtime introspection.
- `cyberos.design.get_token(name)` — single-token lookup.

These are useful when an agent needs to show a Member "what does our brand look like" without rendering a UI; they are not used by component code.

## Alternatives Considered

- **Skip the design system; let modules style themselves with Tailwind defaults.** Rejected: visual drift across 22 modules is the single most predictable failure mode; the lint plugin is the architectural control.
- **Use Material UI / Chakra / shadcn-ui as the component library and only ship tokens.** Rejected: Vietnamese typographic correctness, the Genie panel's bespoke chips, and the brand-colour discipline are not satisfiable from a generic library out of the box. We may *adopt* shadcn-ui's headless primitives under the hood (Radix UI), but the public surface is `@cyberskill/components`.
- **Single mega-package combining tokens + components + docs.** Rejected: tokens have stricter version stability than components; modules should be able to bump components without bumping tokens (which would force a synchronised update across all modules).
- **Defer the Storybook site to P1.** Rejected: without a docs site, every module's UI review devolves into design-by-Slack and module owners pick the wrong primitive.

## Success Metrics

- **Primary metric.** S0-1 → S0-6 demos pass: (1) the host shell + GENIE remote + every P0 module's UI consume the published `@cyberskill/tokens` v1.0.0; (2) Storybook is live at the canonical URL; (3) the lint plugin runs in CI for every module repo; (4) every component passes axe-core AAA contrast tests on default token pairings; (5) Vietnamese-typography do/don't gallery shows correct diacritic rendering across all components.
- **Adherence metric.** Hardcoded-hex / hardcoded-spacing lint violations across all module repos = 0 by P0 exit.
- **Bundle metric.** Each module's initial JS bundle ≤ 50 KB gzipped (PRD §7.2 "Module Ready"); the components import is tree-shakeable so per-component import is measurable.
- **A11y metric.** Every default token pairing (text on surface) meets WCAG 2.1 AAA contrast; lint catches custom pairings.

## Scope

**In-scope (S0-1 base + ongoing through S0-6).**
- npm workspace `cyberskill/design-system` with `@cyberskill/tokens` + `@cyberskill/components` + `@cyberskill/eslint-plugin-tokens-only`.
- Token JSON authored in W3C DTCG format with the eight categories above.
- Self-hosted Be Vietnam Pro + JetBrains Mono woff2 in the tokens package.
- Genie-tokens sub-package versioned with persona.
- ~50 React 19 components covering layout, typography, form, data, navigation, feedback, and domain primitives.
- Storybook docs site at `design.cyberskill.world` (private).
- ESLint plugin enforcing tokens-only.
- Figma sync.
- Two read-only MCP tools.
- Migration guide for any future major bump.

**Out-of-scope (deferred).**
- iOS / Android format token outputs (P3 mobile).
- Public access to the Storybook site (P4).
- Per-tenant theme overrides (P3 — when external tenants exist; PRD §14.4.1).
- Animations beyond static lottie state files for the mascot (P2).
- Email-template tokens (P1 EMAIL FR cluster will extend).
