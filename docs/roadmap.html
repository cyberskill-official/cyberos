Gap to 100% (complete + audited + locally deployable)

The P0 code is complete. What stands between here and a clean local test run is the MCP DB slice (the one change since the last full green bring-up on 2026-06-25) plus a fresh end-to-end verification. In order:

  1. operator Merge auto/mcp-db-slice into main.
  2. operator Bring up the local stack (services/dev/docker-compose.yml) and apply every migration, including mcp-gateway 0013-0017 (add mcp-gateway to the migrate loop after auth).
  3. operator Put a real MCP_KMS_KEY (base64 of 32 bytes) in the local .env; required for the sealed elicitation / task payload path.
  4. verify Run every module suite (auth with --skip create_subject_p95, memory, email, proj, obs-*, mcp-gateway, ai-gateway) plus the MCP DB-path integration tests, plus the end-to-end chat smoke (step 6 of the local-dev guide).
  5. done Independent audit of the DB-slice diff - complete 2026-06-28. Verdict: logic sound. Four hardenings applied and green: gate fail-closed on unresolved caller, caller-scoped confirmation verdict, boot fail-closed without KMS, doc honesty. One go-live item recorded (repo-wide DB role grants, see deploy/vps/README).
  6. done FR status reconcile - applied. obs 001/003/005/006/007/008/009 moved to implementing; mcp left honest (003 + 004 are done-candidates pending the gate run).

After these, P0 is 100% complete, audited, and locally test-ready. Go-live (docker-validate deploy/vps/ compose + Caddyfile, rotate any in-tree secrets) is the separate next phase.
