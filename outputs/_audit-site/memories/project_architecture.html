Cross-cutting architecture (high-level)

Layer cake

CyberOS is six logical layers stacked from edge to data:

Edge — Cloudflare (CDN, TLS termination, DDoS, WAF). One logical instance.

Gateway — GraphQL Federation (Apollo v2 supergraph), MCP Gateway (2025-11-25 spec), AI Gateway (model routing/caching/redaction), Host Shell (Module Federation). One of each.

Modules (subgraphs) — per-domain logic, GraphQL schema, MCP tools. Four in P0 (AUTH, BRAIN, GENIE, CHAT). 22 by P4.

Data — PostgreSQL 17 with extensions (pgvector for HNSW vectors, Apache AGE for graph, PGroonga for multilingual full-text), Redis (caches/queues/rate limits), NATS JetStream (events/streams/replay), S3-compatible object store (Hetzner S3, MinIO), HashiCorp Vault (key hierarchy). One cluster per region.

AI Substrate — BRAIN (three-layer per tenant), LangGraph runtime, persona registry. One logical per tenant.

Infrastructure — Kubernetes, ArgoCD GitOps, LGTM observability stack. Per-region.

Key cross-cutting choices

GraphQL Federation v2 (Apollo). Subgraph per module; supergraph composed at the gateway. Schema check enforced on subgraph publish.

Module Federation (frontend). Each module is a remote bundle composed by the host shell at runtime. Initial JS budget per module ≤50 KB; heavy charts/editors lazy-loaded.

MCP Gateway (2025-11-25 spec). OAuth 2.1 + PKCE; PRM (.well-known) discovery; per-module MCP servers federated through one endpoint; tool-call audit trail. External MCP clients (Claude.ai, Cursor) call CyberOS tools — never the other way around.

AI Gateway. Centralised model routing across providers (Bedrock primary, Anthropic and OpenAI Zero-Data-Retention fallbacks). Latency budgets enforced. Explanation pane standardised. Caching and redaction at the gateway, not in modules.

Auth & RBAC. RBAC + Scope Contract Guard (SCG) over ABAC. Roles: Founder/CEO, Engineering Lead, Account Manager, HR/Ops Lead, Member, Auditor, DPO. Both human and agent callers go through the same RBAC.

Multi-tenancy. tenant_id on every entity; Postgres RLS as the primary isolation; schema-per-tenant + RLS combination at the storage layer; per-tenant region pinning for residency. Single-tenant simplifications through P3 (one tenant, plumbing in place).

Audit & event log. Merkle-chained audit log (DEC-019) for the BRAIN filesystem layer; NATS JetStream for module events on the canonical subject pattern `{module}.{entity}.{verb}`.

Tenant deployment. Hetzner primary; Vultr Singapore; OVH EU at P3+. Cloudflare edge.

How to use this memory

Use this when scoping a new module, troubleshooting a cross-cutting concern, or evaluating whether a proposed architecture change conflicts with foundational decisions. The full DEC catalogue is referenced from `memories/decisions/DEC-001-pointer-to-prd-dec-catalog.md`.
