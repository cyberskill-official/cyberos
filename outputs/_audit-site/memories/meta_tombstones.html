Tombstone registry

One line per soft-deleted memory: `<memory_id> <audit_id> <reason>`. Tombstones are never erased; the body of the original file is kept verbatim, only its frontmatter flips to `tombstoned: true` plus `deleted_at`/`deleted_by`/`tombstone_reason`. Hard-erase is handled separately by a human-driven right-to-erasure flow.

How to use this memory

When you tombstone a memory file, append exactly one line below this section. Do not edit existing lines.
