.cyberos-memory

This directory is the universal memory store for the CyberOS project, governed by the protocol in `docs/CyberOS-AGENTS.md`. It is the lossless, append-mostly, deterministic source of truth that any AI agent working in this project reads on session start and updates on every meaningful change. Everything outside `audit/` may be edited or augmented by an authorised agent or human; tombstoning is a soft delete (frontmatter flag plus `meta/tombstones.md` registry), never an erase.

Do not hand-edit `audit/*.jsonl`. The audit ledger is append-only and Merkle-chained; in-place edits break the chain and freeze writes. To correct a prior row, append a new `op:"corrects"` row that references the bad `audit_id` and `chain`. Manifest updates are made via the `str_replace` operation so they get audited like everything else.

Portable export filename pattern: `memory-export-<YYYY-MM-DD>-<scope>.zip`. Bundles include `manifest.json`, `README.md`, all scope folders, the full `audit/` chain, and active `conflicts/`. Excluded from bundles: `index/`, `.lock`, `exports/`. Verify chain via `audit/`.
