REF-031 — Protocol update policy (§0.5)

Real-world trigger

User question (Stephen, 2026-05-06): *"AGENTS.md behaves like global instructions when copied to local machine. Is there any way to force-sync it with CyberOS's AGENTS.md to make sure all distributed BRAINs are updated when CyberOS has a new BRAIN version?"*

The agent's diagnosis surfaced two pre-existing gaps:

1. AGENTS.md was silent on its own update flow. No tripwire for hand-edits, host-platform silent updates, or accidental drift.

2. "Force sync" as proposed would defeat §0.2 — the consent gate that protects from prompt injection would also block the forced sync; auto-applying upstream protocol changes is itself a §0.2 sev-0 violation.

Refinement adopted

TIER 1 (SHA pinning + forward-compat tripwire) + TIER 3 (additive-only schema rule + rollback path). TIER 2 (signed CyberSkill releases via MCP) deferred to BRAIN-module P1 ship.

What changed in AGENTS.md

  • New §0.5 (Protocol update policy) — sev-0 — defines canonical SHA computation, manifest pin, session-start tripwire, approval phrase, rollback flow, signed upstream release flow, bootstrap behaviour, §0.2 forbidden list.
  • §6 manifest example gained `protocol` block (sha256, approved_at, approved_by, loaded_path, signing_keys, last_checked_at).
  • §7.1 op enum gained `protocol_upgrade` and `protocol_rollback`.
  • §13.0 state classifier gained `INCOMPATIBLE:protocol-sha256-mismatch` row.
  • §13.1 bootstrap gained step 12 (auto-pin) and step 13 (seed `meta/protocol-history/`).

    • Why this is the right fit

    The trio of authenticity (signatures), authorization (chat-turn approval phrase), and auditability (`op:"protocol_upgrade"` rows + `meta/protocol-history/` archive) maps cleanly onto the existing protocol's discipline. SHA-pinning piggy-backs on the existing canonical-JSON discipline (§7.2). TOFU trust establishment matches the SSH model that's well-understood. Bootstrap auto-pin keeps friction zero for solo use today.

    Trade-offs

    Accepted three minor frictions:

  • Edits to AGENTS.md without the §0.5 phrase trigger `INCOMPATIBLE:protocol-sha256-mismatch` and freeze writes. Mitigation: the diff is surfaced; the user reverts or approves.
  • Bootstrap requires `meta/protocol-history/` even when not yet used. Cost: a `.keep` file. Benefit: the upgrade flow has nowhere to fail later.
  • The cyberos repo must bake CyberSkill upstream signing fingerprint into `cyberos/.protocol-signing-key` to make first-clone seamless. Cost: one tracked file. Benefit: zero-touch trust establishment for new employees.

    • How to use this memory

    When proposing or evaluating future protocol-distribution refinements, read this REF + DEC-094 together. Future signed-release infrastructure (TIER 2) lands as a follow-on REF when the BRAIN module is being built.
