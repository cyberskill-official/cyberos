REF-022 — Diagnostic-verb carve-out for PRISTINE auto-bootstrap (§1.2 + §13.0)

Real-world trigger

The first downstream consumer of the post-Bundle-F protocol — a fresh Cowork session at `/Users/stephencheng/Projects/CyberSkill/sale-noti/` — ran `healthcheck` against a PRISTINE BRAIN. The agent there correctly reasoned that bootstrapping silently would change the diagnosed state and held off, surfacing the friction as an §0.4 candidate.

This is the first refinement originating from real downstream use rather than meta-protocol design. The pattern is significant: the protocol has reached the point where its operational discipline (§0.4 standing rule, §1.10 verify-before-respond, the propose-then-adopt loop) is firing correctly on real first-encounter behaviour by fresh agents.

Refinement adopted

TIER 1 only. Two-line carve-out clause + new §13.0 row + manifest.health_check_policy.diagnostic_verbs[] extension. No TIER 2 / TIER 3 because the gap is narrow and well-bounded.

What changed in AGENTS.md

  • §1 step 2 gained a "UNLESS … recognised diagnostic verb" clause naming the default verb list and pointing at the manifest override.
  • §13.0 gained `PRISTINE-DIAGNOSTIC-HOLD` row.
  • §6 manifest gained `health_check_policy.diagnostic_verbs[]`.

    • Why this is the right fit

    The pre-G silent-auto-bootstrap directive was correct for task-oriented first-use; the carve-out narrows it for diagnostic first-use only. The default verb list is conservative (literal matches only); the manifest override lets projects tune. The carve-out preserves §13.0's general PRISTINE → bootstrap pattern; it just defers it under one specific condition.

    The §0.6 related-files rule was applied: CHANGELOG entry added; README Part 8 gained Bundle G as the eighth real-world trigger; this REF was written.

    Trade-offs

  • The verb list lives in two places (§1 prose for bootstrap defaults; manifest for runtime overrides). Drift risk is small in practice.
  • "Status" is a common word that may trigger the carve-out spuriously. One-line consent resumes; not a major friction.

    • How to use this memory

    When designing future protocol refinements that should land upstream because they originated downstream, REF-022 is the canonical pattern: downstream agent surfaces the friction inline → user approves upstreaming → upstream cyberos lands the bundle → downstream pulls via next §0.5 upgrade. Bundle G is the first instance.

    When extending the diagnostic-verb list (e.g., for non-English projects that use Vietnamese), update the project's `manifest.health_check_policy.diagnostic_verbs[]` rather than amending §1 step 2's prose — the prose default is universal, the manifest override is per-project.
