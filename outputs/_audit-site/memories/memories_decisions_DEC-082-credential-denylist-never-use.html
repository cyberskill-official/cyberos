Sharpened credential denylist: never store AND never use (AGENTS.md §9.3)

Extend §9.3 secrets rule from "never store" to "never store AND never USE." When user pastes password/token/recovery phrase/OAuth/API key/private key in chat — even with explicit instruction — agent MUST: refuse to USE for authentication; prefer connected MCP/OAuth flows; recommend rotation in the same response. Trio "never store" + "never use" + "advise rotation" is non-negotiable. Real validation: Notion password incident 2026-05-04 — agent refused use, used MCP, advised rotation.
