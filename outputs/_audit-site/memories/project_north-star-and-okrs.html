North-star metric, strategic goals, OKRs

North-star

Percentage of routine business actions performed by AI agents (CUO + MCP), weighted by minutes-saved. Computed per action class — for each of {create_task, submit_timesheet, send_email, draft_payslip_narrative, log_crm_activity, schedule_meeting, summarise_thread, propose_invoice, generate_status_update, …}, the system measures (a) volume initiated by Genie/CUO/MCP versus volume initiated by human UI, and (b) median minutes-saved per action class via internal time-and-motion baseline. Dashboard published in OBS, reviewed weekly by the founder.

Strategic goals (G1–G8)

G1 — Operational truth. One system of record for every CyberSkill business event by P1 exit.

G2 — Agent parity. Any task a human can do in CyberOS, an AI agent can do via MCP, with the same RBAC, by each module's "Module Ready" criterion.

G3 — Modular ownership. Each module is owned end-to-end by a single role.

G4 — Compensation fidelity. REW / LEARN / ESOP encode the legal Appendix exactly. P1-protection invariant, anti-retroactive parameter versioning, and Good/Bad Leaver branches enforced at the system level.

G5 — Universal memory. BRAIN indexes every eligible module write within p95 ≤5s; CUO and agents always answer with cited sources or "I don't know"; the filesystem layer is portable and signed; conflicts surface for resolution.

G6 — Commercial readiness. Multi-tenant, billable, brandable by P4 exit.

G7 — Cost discipline. Production infra ≤$380/month at internal P2 scale; ≤$2,200/month at 50-tenant scale.

G8 — Founder cognitive load reduction. CUO reduces founder hours-per-week spent on ops/admin/exec-thinking by ≥40% by P1 exit, validated against TIME entries.

Headline OKR targets

P0 exit — DAU on CHAT vs Slack/Zalo ≥9/10 Members; daily Genie/CUO interactions/Member ≥5; BRAIN chunks indexed ≥10k; ≥80% of Members have edited their `.cyberos-memory` at least once.

P1 exit — % CyberSkill weekly ops captured in CyberOS ≥90%; time-tracking entries/Member/week ≥20; ambient nudge signal-to-noise (% accepted vs dismissed) ≥60%; Members receiving payslips through REW ≥10/month for ≥3 cycles.

P2 exit — % routine tasks initiated via MCP/CUO ≥30%; first SP valuation cycle Board-signed.

Continuous from P0/P1 — Successful AI tool calls/total ≥95%; CUO citation rate ≥98%; recompute test on stored payslips 100% identical re-runs; platform availability (28-day rolling) ≥99.5%; p95 GraphQL ≤400 ms; p95 CHAT message-deliver ≤200 ms; p95 CUO text response ≤2 s; p95 brain.search ≤250 ms.

Guardrail (anti-)metrics

Memory containing a denylist hit (compensation, gov ID, bank, health PII, secrets) → 0 tolerance, structural rejection. CUO autonomous external action attempts → 0 (any attempt is an incident). Persona drift (CUO output deviating from the dual-signed persona spec) → caught by automated drift detection (DEC-037), feature-flag-paused on detection.

How to use this memory

Use this when answering "are we on track" questions, when prioritising work that has weak product justification but strong metric impact, or when proposing a new metric. Adding or changing a key result requires founder approval and a dual-signed audit entry.
