REF-024 — Compact §14 format gated by operational_mode

Real-world trigger

Stephen, post-Bundle-H: *"sometime this section so long and hard to read, is there any way to present it more verbose & human easier read?"* (Yes, the user said "more verbose" — but in context they meant "more readable / scannable", not literally more lines.) Survey of prior turn outputs confirmed: every §14 block was ~14 lines, ~9 reading "no change" verbatim. Signal-to-noise poor for daily use.

Refinement adopted

TIER 1 only: split §14 into two formats keyed by `manifest.operational_mode`. No new mechanism; reuses existing field.

What changed in AGENTS.md

  • §14 opening paragraph rewritten to declare the two-format split.
  • §14.1 (new): compact format — `Δ Changes` / `Status` / `unchanged` three-block layout.
  • §14.2 (new): full format — pre-I per-scope-explicit format with `maintenance` banner addition.
  • §14.3 (renumbered from prose paragraph): coverage stat for ingestion ops; mandatory in both formats.
  • §14.4 (new): authority clarifier — audit ledger is authoritative; §14 is human-summary only.

    • Why this is the right fit

    The user's friction is real and observable. The fix reuses an existing field (`operational_mode`) instead of inventing new machinery. Compact format defaults match the "daily use" context; full format defaults match the "protocol developer" context. Both formats preserve the §14.3 coverage stat that's load-bearing for ingestion-op honesty.

    The audit ledger remains the chain-recovery surface; the §14 block is courtesy summary. So changing the rendering can't break audit integrity — only changes what humans see in chat.

    Trade-offs

  • Compact format's `unchanged:` roll-up loses per-file granularity. Acceptable: protocol developers can flip to `verbose` mode any time.
  • Two formats means future agents support both. Acceptable: the §14 block is human-facing; future agents primarily read the audit ledger.

    • How to use this memory

    When the user surfaces UX friction with the §14 block, check whether they're running `operational_mode: normal` (default → should already be compact) or `verbose`. If `verbose`, ask whether they want to switch to `normal`. If `normal` and they still find it long, that's a Bundle-J-or-later candidate.

    When designing future protocol changes, update both §14.1 and §14.2 examples to keep them in sync.
