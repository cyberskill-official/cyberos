Module catalogue (22 modules)

Each module is one Apollo Federation v2 subgraph + one Module-Federation frontend remote, owned end-to-end by exactly one role. Listed in phase order with depends-on and value statement.

P0 — Foundations (7 modules)

AUTH — Authentication & Authorization. Owner: Engineering Lead. Identity, RBAC, SSO via Google Workspace; agent-equal auth. Foundation for every other module.

AI — AI Gateway. Owner: Engineering Lead. Depends on AUTH, OBS. Centralised model routing + caching + redaction across Bedrock primary + Anthropic/OpenAI ZDR fallbacks.

MCP — Model Context Protocol Gateway. Owner: Engineering Lead. Depends on AUTH, AI. Single MCP endpoint federating per-module servers; tool registry; OAuth 2.1 + PKCE; PRM discovery.

OBS — Observability. Owner: Engineering Lead. Depends on AUTH. LGTM stack — logs, metrics, traces, AI usage, audit. Hosts the Compliance Cockpit.

CHAT — Internal Chat. Owner: Engineering Lead. Depends on AUTH, AI, BRAIN. Mattermost fork; AI-native features; Vietnamese full-text search via PGroonga; Slack/Zalo replacement.

BRAIN — Universal Knowledge Layer. Owner: Engineering Lead. Depends on AUTH, AI. Three-layer memory (filesystem .cyberos-memory + vector/graph + archival corpus); tenant-scoped; cited retrieval.

GENIE — Genie / CUO AI Assistant. Owner: Founder/CEO + Engineering Lead. Depends on AUTH, AI, MCP, BRAIN. Persistent CUO persona with 10 C-level skills + 4 emergent; ambient nudges via Notify/Question/Review.

P1 — Internal Productivity (8 modules)

PROJ — Project Management. Owner: Account Manager. Depends on AUTH, BRAIN, CHAT. Linear-style three-primitive PM (Issue/Cycle/Project); sync-engine.

TIME — Time & Expense. Owner: Account Manager. Depends on AUTH, PROJ. Time entries, billable rules, expense capture; Genie auto-categorisation.

CRM — Client Management. Owner: Account Manager. Depends on AUTH, BRAIN, CHAT. Pipeline, accounts, contacts, deals; agent-loggable activity; confidence-bands not people-scoring.

KB — Knowledge Base. Owner: Founder/CEO. Depends on AUTH, BRAIN, AI. Living docs ingested into BRAIN; AI Q&A grounded; GraphRAG navigation.

HR — Human Resources. Owner: HR/Ops Lead. Depends on AUTH, BRAIN. Profiles, leave, contracts, sabbatical accrual.

EMAIL — Internal Email & Shared Inbox. Owner: Account Manager + Engineering. Depends on AUTH, BRAIN, CHAT. Stalwart-based JMAP+IMAP+SMTP; CaMeL anti-injection; Missive-style UX.

REW — Total Rewards (Compensation). Owner: HR/Ops Lead + CEO. Depends on AUTH, HR, TIME. Faithful 3P income + BP fund + anti-retroactive parameter versioning. Treated as EU AI Act Annex III §4 high-risk.

LEARN — Learning & Promotion. Owner: HR/Ops Lead + CEO. Depends on AUTH, HR, REW. VP roll-up; Hội đồng Chuyên môn (Professional Council) workflow.

P2 — Operations (2 modules)

INV — Invoicing. Owner: Account Manager. Depends on AUTH, CRM, TIME. Invoice lifecycle; AR aging; Stripe / Wise / VND-PSP integration.

ESOP — Phantom Stock. Owner: CEO + Board. Depends on AUTH, HR, REW. Grants, vesting, valuation, put options, Good-Leaver / Bad-Leaver branches.

P3 — SaaS Readiness (2 modules)

RES — Resource Planning. Owner: Founder/CEO. Depends on AUTH, HR, PROJ, TIME. Capacity vs forecast; allocation Gantt.

OKR — OKR / Strategy. Owner: Founder/CEO. Depends on AUTH, BRAIN. Quarterly OKR cycle; cascade through teams.

P4 — Client-Facing (3 modules)

DOC — Document Signing. Owner: Account Manager + Legal. Depends on AUTH, KB, EMAIL. eIDAS QTSP integration; AATL adv-cert; archival.

PORTAL — Client Portal. Owner: Account Manager. Depends on AUTH, PROJ, INV, DOC. Branded customer-facing surface.

TEN — Tenancy & Billing (External). Owner: Founder/CEO. Depends on all P0–P3 modules. Tenant signup, billing, plan management, brandability.

Module Ready definition (CI-gated at every phase exit)

Owner identified in code-OWNERS file. Apollo Federation subgraph published; schema check passes. Module-Federation remote URL listed in host shell registry. MCP server endpoint listed; tools annotated; tests passing. Module publishes its own version, phase, and module-readiness flag via `brain.module_health`. Functional requirements named under FR-{MOD}-{NNN}. RLS policies, residency, egress filter respected. NATS events emitted on canonical subject `{module}.{entity}.{verb}`. TestExperiment subgraph for evals + a hosted demo workspace. Initial JS bundle < 50 KB; heavy charts/editors lazy-loaded.

How to use this memory

Use this when scoping work, identifying owners, or planning dependencies. For per-module detail (FR-{MOD}-{NNN}, MCP tool surfaces, GraphQL contracts), see `docs/CyberOS-PRD.docx` Part 9 and `docs/CyberOS-SRS.docx` Part 7. Do not duplicate that detail in the BRAIN — link to it.
