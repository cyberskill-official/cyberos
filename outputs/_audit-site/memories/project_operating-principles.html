Operating principles (non-negotiable)

Eight principles govern every architectural and product decision in CyberOS. The last two are explicitly upgraded by the SRS and marked with a star.

AI-native from day one. Every module exposes a native MCP server with the same RBAC humans use. Genie/CUO is omnipresent across every UI surface. There is no module where AI is bolted on after the fact.

Modular plug-in architecture. Every module is an independently deployable Apollo Federation v2 subgraph plus a Module-Federation frontend remote, owned end-to-end by exactly one role. Module readiness is gated by a checklist (subgraph published, MCP tools registered, NATS events emitted, RLS enforced, bundle ≤50 KB initial JS).

Internal-first. CyberSkill is the only tenant through P3. External tenant signup, billing, and Tenant Admin UX are P4 only. Multi-tenancy plumbing is in place from P0 (tenant_id everywhere, RLS policies) but only one tenant exists for 12+ months.

Compliance-by-construction. Vietnam home regime (PDPL Law 91/2025 + Decree 356) is the cornerstone. Per-tenant residency, A05 filings, mandatory DPO, AI-derived-data-as-PD treatment, and Trust Center are P0 deliverables — not retrofits.

Compensation honours the social contract. REW / LEARN / ESOP encode the legal Total Rewards Appendix faithfully. The P1-protection invariant ("evaluation never reduces base salary in cash"), anti-retroactive parameter versioning, and Good Leaver / Bad Leaver branches are hard system properties, not policies that can be bypassed by an admin.

Universal memory, three layers ★. BRAIN is a layered architecture: a Claude-style filesystem `.cyberos-memory` at the local edge, a Mem0-style vector + graph layer in the middle, and a Letta-style archival corpus at the cold tier. Every layer enforces the compensation/equity exclusion via the ingestion-side denylist (DEC-036).

Proactive but never autonomous ★. CUO observes module events and surfaces context through Notify / Question / Review primitives; it never auto-acts on irreversible operations (sending external comms, signing contracts, moving money, terminating accounts). Microsoft Recall is the cautionary tale; LangChain ambient agents is the model.

Agent parity. Any task a human can do in CyberOS, an AI agent can do via MCP under the same RBAC, audit, and tenancy. The MCP tool surface may be a strict subset of the UI but is never a superset. This inverts the value proposition from "buy the tool, hire people to use it" to "buy the tool, the AI uses it for you."

How to use this memory

When evaluating a feature proposal or architectural change, run it past these eight principles. Anything that violates one of them — even with a strong product reason — needs an explicit, dual-signed exception (founder + Lead Architect). Drift on these principles is the failure mode CyberOS is designed to prevent.
