DEC-106 — §14 omission rule + audit-trail suppression in normal mode (Bundle N TIER 1+2)

Decision

The §14 end-of-response block is now **conditional in normal mode** rather than mandatory. Two new sub-sections govern the conditionality:

  • **§14.0 (sev-2)** — omit the entire §14 block when ALL of: (a) `manifest.operational_mode == normal`, (b) no `op:rejected|revert|warn|health_check` row this turn, (c) no memory mutations this turn (audit-row count ≤ 2 and only `session.start`/`session.end` bookends), (d) most-recent §8.7 self-audit reports 0 CRITICAL and 0 WARN.
  • **§14.1.1 (sev-2)** — when §14.1 compact IS emitted in normal mode, omit the `audit/<YYYY-MM>.jsonl: <N rows appended; head=sha256:…>` line unless a finding (rejection/revert/warn) occurred this turn or the most-recent §8.7 reports issues.

    • `verbose`/`debug`/`maintenance` modes always emit the §14.2 full block including the audit-trail line — visibility outranks compactness in development sessions.

    Why

    User feedback (cowork-session 2026-05-10): the §14 block plus the audit-trail one-liner was flooding chat on every turn, even when the chain was healthy and nothing meaningful changed. Direct quote: *"show Audit trail after each messages make the conversation flooded, just show in maintenance mode or when issues arise"* and *"can we compress 📝 .cyberos-memory updated section more? show full verbose in maintenance mode, but only show changes summary on normal (default), if no issues arise don't need to show memory changes, just show other files' changes"*.

    The audit ledger remains the authoritative record (§7); chat-side silence does not affect chain integrity. Users can run `runtime/tools/cyberos_validate.py` or the configured `health_check_policy.on_demand_phrase` (default *"run BRAIN healthcheck"*) on demand.

    Authority

    Adopted via §0.5 chat-turn approval phrase on 2026-05-10:

    > approve protocol upgrade to sha256:8060fe2e188e1793e9dbc758b34a8198617ff8bf8a3320a2012595faf3012dab

    Trade-offs

  • **Pro**: signal-to-noise improves dramatically — most turns produce no §14 block at all, so when the block DOES appear, it carries information the user actually needs.
  • **Pro**: the audit-trail one-liner returns automatically when something goes wrong; no manual mode-switching needed for diagnosis.
  • **Con**: agents that don't track per-turn rejection/revert/warn counts may incorrectly omit when they shouldn't. Mitigation: the four §14.0 conditions are narrow and cheap to check; the §8.7 self-audit catches schema drift if any agent omits incorrectly.
  • **Con**: the prose for the new rules is +1922 chars in AGENTS.md, slightly larger than what's saved per turn — but the savings compound across hundreds of future responses.
