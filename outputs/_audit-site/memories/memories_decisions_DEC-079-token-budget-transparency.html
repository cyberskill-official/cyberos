Token-budget transparency on >500-line sources (AGENTS.md §4.10.2 (was §4.11 pre-Bundle-M))

Before processing any source >500 lines or >50 KB, agent MUST declare chunking plan in response ("Source is N lines / M KB. Reading in K chunks of ~Y lines each. Tracking high-water mark."). After all chunks process, before writing the digest, agent MUST confirm coverage ("All K chunks processed; coverage P/N lines = R%."). Skipping the announcement is itself a §4.10 violation.

How to use this memory

Frozen decision. Cite when answering questions about CyberOS-AGENTS.md v1.1.0 or when implementing the BRAIN service.
