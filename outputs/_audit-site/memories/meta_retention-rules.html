Retention rules

Each memory file's `retention.rule` field references one of the rules below. `retention.earliest_delete` may be set to an ISO date past which the memory is eligible for tombstoning during consolidation. Tombstones are soft deletes and never erasures (§4.6).

`personnel-default-3y` — personnel-classified memories. Earliest delete = `created_at + 3 years`.

`client-default-7y` — client-classified memories. Earliest delete = `created_at + 7 years` or end-of-engagement + 7 years, whichever is later.

`operational-default-1y` — operational-classified memories. Earliest delete = `created_at + 1 year`.

`public-no-expiry` — public-classified memories. Indefinite retention.

`legal-hold-indefinite` — overrides the default rule when the memory is subject to litigation or regulatory hold; lifts only via explicit human action.

How to use this memory

Pick the rule that matches the memory's classification at create time. If the memory transitions classification later, update both `classification` and `retention.rule` together. CyberOS-specific compensation retention (10 years per DEC-020) lives in the REW module — do not duplicate that policy here.
