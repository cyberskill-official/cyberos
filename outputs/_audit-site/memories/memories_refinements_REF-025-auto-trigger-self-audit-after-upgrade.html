REF-025 — Auto-trigger §8.7 after `protocol_upgrade` + uppercase BRAIN

Real-world trigger

Stephen, post-Bundle-I: *"can we auto trigger scan and re-arrange/refine the .cyberos-memory after AGENTS.md update, because there maybe breaking changes or rules that need to adapt, and how to manual trigger that?"* Plus immediate reinforcement on the manual phrase: *"for manual i want 'run BRAIN healthcheck' instead"*.

Two distinct concerns in one bundle: (1) wire §8.7 into the post-upgrade flow automatically; (2) finish Bundle H's uppercase-BRAIN consistency in default trigger phrases.

Refinement adopted

TIER 1 covers both. §0.5 step 4 + §6 manifest extension (post_upgrade_phrase) + §8.7 Post-upgrade subsection + §1 step 2 verb-list update + cyberos manifest re-pin with new defaults.

Why this is the right fit

The §8.7 schema-validate check ALREADY catches new-schema-failures; the missing piece was just automatic invocation after `protocol_upgrade`. One-paragraph amendment — minimal mechanism, high migration safety.

The uppercase-phrase fix completes Bundle H's case-sensitivity discipline. Three default phrases (`on_demand_phrase` and three `diagnostic_verbs[]` entries) still used lowercase "brain"; Bundle H made strict-uppercase the rule but left the defaults in §6 stale. Bundle J fixes this.

What changed in AGENTS.md

  • §0.5 approval-phrase flow: added step 4 (auto-trigger §8.7) with explicit skip phrase.
  • §6 manifest defaults: `on_demand_phrase` lowercase→uppercase; new `post_upgrade_phrase` field with default *"rescan BRAIN"*; `diagnostic_verbs[]` entries that mention BRAIN switched to uppercase.
  • §1 step 2 verb list: matches new manifest defaults; one-sentence note about uppercase consistency with §0.3.
  • §8.7: new "Post-upgrade scan" subsection distinguishing report naming and trigger sources.

    • Trade-offs

  • Auto-trigger adds latency to every `protocol_upgrade` — acceptable; migration safety is worth a sub-second pass.
  • Project-level overrides not migrated — by design; per-project autonomy.
  • Two report-file naming conventions — minor; provenance traceability worth it.

    • How to use this memory

    When the user approves a `protocol_upgrade`, run §8.7 automatically as step 4 of the §0.5 flow. Surface findings. If the user says *"rescan BRAIN"*, manual-invoke the same scan. The skip phrase *"skip post-upgrade scan"* is the only valid bypass — log it as `op:"skipped-by-user"`.

    When designing future protocol changes, predict migration impact: which existing memories will fail the new schema? Surface this to the user during Bundle proposal so they understand the post-upgrade scan's expected findings.
