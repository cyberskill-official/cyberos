REF-023 — Strict uppercase BRAIN alias (§0.3)

Real-world trigger

Stephen, post-Bundle-G review: *"i notice that 'brain' still work? i want only 'BRAIN' will be understand as the memory, because some topic relate to human brain may trigger too, right?"*

The §0.3 alias was designed for unambiguous memory-store references but its wording didn't enforce case-sensitivity. False-positive risk: anatomy / metaphor / topic uses of lowercase "brain" could trigger memory-store actions inappropriately.

Refinement adopted

TIER 1: §0.3 amended with case-sensitivity clause + lowercase-is-normal-language clarifier + ambiguity-disambiguation question rule. No TIER 2/3 needed.

What changed in AGENTS.md

  • §0.3 opening sentence now explicitly states "(literal uppercase B-R-A-I-N; case-sensitive — lowercase 'brain' does NOT trigger this alias)".
  • New paragraph between opening and examples list: lowercase 'brain' interpreted as anatomy / metaphor / topic; surface clarifying question when context is ambiguous.

    • Why this is the right fit

  • Closes the observed false-positive surface without breaking any existing usage (Stephen and the agent already use uppercase "BRAIN" consistently in this conversation).
  • Independent of §1 step 2 diagnostic-verb list (Bundle G), which uses lowercase verb intents — the two mechanisms don't conflict.
  • Aligns with the §0.4 propose-then-adopt loop: real-world friction → narrow refinement → upstream so future projects don't re-encounter.

    • Trade-offs

  • Users must type uppercase BRAIN. Acceptable — the alias is a deliberate signal, not an accidental match.
  • Marginal friction on ambiguous lowercase ("what does my brain say?"); resolved by a one-line clarifying question.

    • How to use this memory

    When designing future refinements that affect agent matching/parsing, prefer strict-form rules (case-sensitive, exact phrasing) with explicit ambiguity-disambiguation paths. Bundle H is the canonical example: small rule change, big false-positive-surface reduction, no breakage.
