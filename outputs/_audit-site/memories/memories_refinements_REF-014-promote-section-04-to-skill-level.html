REF-014 — Promote AGENTS.md §0.4 from protocol-level to skill-level

What changed

AGENTS.md §0.4 "Standing rule — always propose protocol refinements when memory issues surface" is itself **unchanged**. But the same *pattern* is now ported to the skill level via DEC-092:

| Layer | What it self-audits | Where refinements land |

| --- | --- | --- |

| AGENTS.md §0.4 (protocol) | Memory operations: shallow ingestion, retrieval miss, conflicts §9.1 can't resolve | `memories/refinements/REF-NNN-...` (this is one such record) |

| Skill `INVARIANTS.md` (DEC-092) | Skill operations: invariant breaches, anomaly streaks, drift signals | `genie.action_log` `op:self_refinement_proposal` rows + a follow-up `memories/refinements/` entry on adoption |

Same shape: declarative truths → runtime check → human-reviewed diff proposal → tier-prioritised amendment. Different surface.

Why

The §0.4 pattern proved its value over many sessions in the BRAIN's own evolution (REF-001 through REF-011 are all §0.4-driven). Skills were the obvious next surface: a skill in production accumulates real-world behaviour data the same way the BRAIN does. Without a self-audit mechanism, a skill that drifts has to wait for a human to notice. With one, the skill flags its own drift in real time and proposes the fix.

Trigger

User stated as a top-line v0.2.0 requirement: *"audit itself to detect issues/abnormal behavior at runtime and proactive suggest self refinements, so controllers (human users) can do HITL to apply necessary changes, then resume the workflow"*.

The intuition was the user's; the recognition that this IS §0.4 at a different layer was the design move.

Tier

**Tier 1** — without this, skill-level autonomy is aspirational rather than enforceable. Adopted as DEC-092 (the carrier decision).

How to apply

When designing or reviewing a skill:

  • Author 3-8 invariants in `INVARIANTS.md` covering the skill's highest-value contracts. Start with the contract that the skill's CHANGELOG `### Driver` sections cite most often.
  • Each invariant has a Refinement template — what `proposed_diff` the runtime should emit on breach. Make it specific (cite the section, propose the exact prose change). Vague templates produce vague proposals that humans reject.
  • Anomaly signals are the early-warning system. Set thresholds conservatively at first; tighten as the skill stabilises.
  • The auto-refinement loop hands off to manual fine-tune (DEC-093) after 2 proposals on the same theme. That escalation threshold is the line between "the skill can patch itself" and "we need human investigation."
