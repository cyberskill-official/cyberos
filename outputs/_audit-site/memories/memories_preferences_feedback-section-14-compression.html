Preference — §14 silent by default; verbose only on issues; `📁 Files changed:` shows non-BRAIN paths only

Rule

The §14 end-of-response block follows a **three-state triage** keyed on findings AND on whether non-BRAIN files were touched:

  • **Silent (default).** No `📝` header, no horizontal rule, nothing — when nothing meaningful happened in normal mode (no findings, no non-BRAIN file changes, clean self-audit). **BRAIN-only mutations alone DO NOT break silence** — they're agent housekeeping, surfaced via on-demand tooling (`runtime/tools/cyberos_validate.py`, the healthcheck phrase) when the user asks.
  • **Compact.** A single `📁 Files changed:` block listing **non-BRAIN paths ONLY** — when project files (outside `.cyberos-memory/`) changed but no issues arose. No `Δ Changes:` heading, no `Status:` block, no `unchanged:` line, no audit-trail line, no BRAIN paths.
  • **Verbose.** Full output with `⚠️ Findings:` first, then `📁 Files changed:` (non-BRAIN; omit if none), then `Δ Changes (BRAIN detail):` (BRAIN paths; always present), then `Status:`, then optional `Tokens:` — when a finding/rejection/warn/health_check occurred this turn OR the latest §8.7 self-audit reports CRITICAL/WARN OR `operational_mode != normal`. Auto-triggered; no manual mode flip needed.

    • **Why:** the user's mental model is "`📁` = files in MY project". BRAIN-internal mutations (memory writes, audit rows, manifest updates) are agent infrastructure — equivalent to log files. The user shouldn't track them on every turn; they want the chat surface focused on what THEY would have done by hand. The audit ledger preserves full forensic detail for when it matters.

    **How to apply:**

  • Classify each path mutated this turn: BRAIN (`.cyberos-memory/...`) vs non-BRAIN (everything else).
  • Maintain per-turn counters: `finding_count` (`rejected|revert|warn|health_check` ops), `non_brain_change_count`.
  • Evaluate the three-state triage:

    • 1. Findings > 0 OR latest §8.7 has CRITICAL/WARN OR mode != normal → §14.2 verbose.

    2. Else `non_brain_change_count` > 0 → §14.1 compact.

    3. Else → §14.0 omit silently.

  • In §14.1, list ONLY non-BRAIN paths. NEVER BRAIN paths. No qualifier needed (the path itself signals it's outside BRAIN — there are no BRAIN paths in §14.1).
  • In §14.2, list non-BRAIN paths under `📁 Files changed:` (omit block if none) AND BRAIN paths under `Δ Changes (BRAIN detail):` (always present).
  • Never emit a `Tokens:` line until a runtime token counter is wired up via MCP. Approximation via `tiktoken` or character count is forbidden — it would mislead more than help.
  • File paths in `📁 Files changed:` should be project-relative; paths in `Δ Changes (BRAIN detail):` should be BRAIN-relative.
