REF-021 — Comprehensive audit-fix pass + §0.6 related-files update rule

Real-world trigger

User request: *"check whole CyberOS-AGENTS.md content to find things that can be refine/compress/combine/merge/drop..."* (post-Bundle-E). User reinforcement: *"remember always update readme and changelog after AGENTS.md changes."*

A whole-document re-read surfaced 19 issues. The most consequential were silent bugs accumulated across the 2026-05-06 cascade:

  • §5.1 "27 fields" after Bundle A made it 28.
  • §8 "5 phases" after §8.6 + §8.7 made it 7.
  • §8.7 step-4 hash formula contradicted Bundle D §7.2.
  • §4.7 orphan-manifest check would freeze writes after every `protocol_upgrade`.
  • §11.5 step 5 referenced a non-existent §8.5.
  • §9.7 mentioned an undefined "30-day legal hold."

    • Plus 5 stale references (§0 §16 reference, §3 layout missing dirs, §13.1 empty-string vs null, §16 vocabulary, §0.2 schema_version) and 4 compression opportunities (§0.5 redundant paragraph, §4.10 verbose tool list, §4.1/§11.7 overlap, §9.4 project-leak examples).

    Refinement adopted

    All three tiers (TIER 1 bugs / TIER 2 stale / TIER 3 compression) adopted as Bundle F, plus the §0.6 Related-files update rule as a new standing-rule section.

    Why this is the right fit

  • TIER 1 items are factual bugs; fixing them is uncontroversial.
  • TIER 2 stale references compound interest if left; cheaper to fix early.
  • TIER 3 compressions reduce reader friction without removing meaning.
  • §0.6 prevents future drift of the same kind: by codifying "update README+CHANGELOG in the same chat turn as the AGENTS.md edit," doc-level debt cannot quietly accumulate.

    • The §0.6 rule sits at sev-1 (not sev-0) because failing to update related files is recoverable — the protocol stays correct in the meantime, and back-filling works. Sev-0 would freeze writes on forgotten-CHANGELOG, which is friction without value. Standing-rule status (always applied) is what makes drift unlikely without sev-0 enforcement.

    Trade-offs

  • §7.5 adds a small subsection for `corrects` vs `correction_to`; could have folded into §7.1 prose. Trade-off: discoverability vs minimalism. Discoverability won.
  • §8 renumbering: §8.5 now means the explicit "Update manifest" subsection; "phase 5" still means the recompute-manifest action. The naming alignment makes future cross-references unambiguous.
  • §11.7 reduced to a cross-reference: most readers will skip it. Trade-off accepted.

    • How to use this memory

    When proposing protocol changes, treat §0.6 as the cardinal rule for the chat turn — never close a turn that included an AGENTS.md edit without also updating CHANGELOG and README. When auditing AGENTS.md for staleness, the pattern is: re-read every numerical claim ("N fields", "M phases"), every cross-reference ("see §X.Y"), every named feature ("X-day hold", "schema_version"), and every project-specific example. Most stale claims fall into one of these four buckets.
