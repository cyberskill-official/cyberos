CyberOS Glossary

Canonical terminology for the CyberOS protocol + BRAIN.

Audit ledger

`.cyberos-memory/audit/<YYYY-MM>.jsonl`. Append-only, Merkle-chained record of every state change.

Authority hierarchy

`human-edited > human-confirmed > llm-explicit > llm-implicit`. Strict order. Agents may downgrade on uncertainty; never promote.

BRAIN

Whole system: Layer 1 + Layer 2 + Layer 3. Case-sensitive uppercase only per Section 0.3.

Bundle

Named cohort of related protocol amendments shipped together. Single letter (A-Q so far).

Classification

One of personnel / client / operational / public. personnel + client never auto-resolve in conflicts.

Conflict

Two memories in same scope holding contradictory facts. Disputed pairs valid permanent state.

Consolidation

Five-phase pass at session-end. Surfaces candidates, detects conflicts, conservative merge, reorganise, update manifest.

Content gate

Section 4.2 validator. Rejects prompt-injection markers, mixed-script confusables, base64 blobs, control chars, denylisted content.

CUO

Chief Universal Officer. CyberOS's named-brand persona (10 C-suite skills in one).

Denylist

Section 9.3 categorical exclusions: compensation, gov-IDs, bank/card numbers, home addresses, health PII, secrets.

Drift candidate

Section 8.6 signal that a source file's SHA changed since digest was written. Three responses: re-ingest, accept drift, update source.

Gateguard

PreToolUse 3-stage DENY-FORCE-ALLOW hook (Aspect 5.1). A/B tested +2.25 quality improvement.

Manifest

`.cyberos-memory/manifest.json`. Per-project root pointer.

Memory file

YAML frontmatter + Markdown body. 27+1 permitted frontmatter fields per Section 5.1.

Provenance

`provenance.{source, source_ref, confidence}` on every memory.

Refinement

Protocol amendment proposed per Section 0.4. Recorded in `memories/refinements/REF-NNN-<slug>.md`.

Scope

company / meta / module / member / client / project / persona / dm. Determines default sync_class.

Shallow candidate

Section 8.6 signal that a digest's processed_lines/source_lines < 0.80.

Source freshness tier

Integer >= 1; lower = more authoritative. Used in Section 9.1 step-0 conflict resolution.

Subject

Person a memory is about (or owned by). Subjects are sovereign over their own member/.

Supersedes graph

DAG of supersedes/superseded_by pointers. Walks before write to detect cycles.

Sync class

local-only / publishable / shared / client-visible. Section 17.

Tombstone

Soft delete. File body kept verbatim; frontmatter flipped `tombstoned: true`.

Two-phase atomic write

Section 4.4: validate, append audit row, tmp+rename, fsync. Crash-safe.
