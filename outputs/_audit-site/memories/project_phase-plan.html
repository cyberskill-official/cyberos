Phase plan & 12-month milestone arc

CyberOS ships in five gated phases (P0–P4) over an 18–24 month arc. Each phase has a fixed scope, an explicit out-of-phase list, and a hard exit gate the founder must sign.

P0 — Foundations (Months 1–3)

Scope: Hetzner infra across two regions, K8s production cluster, secrets vault. AUTH module with Google Workspace SSO. BRAIN Layer 1 (filesystem `.cyberos-memory` with the six file ops, anti-injection, CRDT sync). BRAIN Layer 2 (vector + graph, hybrid retrieval, GraphRAG community summaries). GENIE/CUO base persona with three skills active (CEO, COO, CTO), Notify class enabled, Review behind feature flag. CHAT module — Mattermost fork integrated, Vietnamese full-text search via PGroonga. Apollo Federation v2 supergraph live with subgraphs for AUTH, BRAIN, GENIE, CHAT. MCP Gateway with the 2025-11-25 spec, OAuth 2.1 + PKCE, PRM discovery. Design system v1.0 in the host shell. OBS basic metrics, traces, and the Compliance Cockpit shell.

Out of P0: EMAIL, PROJ, CRM, KB, HR, REW, LEARN, INV, ESOP, RES, OKR, DOC, PORTAL. Auto-Dream nightly is scaffolding only. Multi-tenancy is single-tenant only (tenant_id plumbed, only one tenant exists).

P0 → P1 exit gate: all 10 employees onboarded into AUTH+CHAT+GENIE; CHAT has fully replaced Slack/Zalo for ≥14 consecutive days; BRAIN Layer 1 has ≥200 captured memories per active employee; Layer 2 ≥1,000 facts; conflict-resolution UI used by ≥3 employees; Genie has produced ≥50 user-accepted drafts in any single week; Compliance Cockpit green on Decree 20 SME regime; zero P0-Sev0 items; clean PEN-test on AUTH and MCP gateway; founder signs the gate-readiness report.

P1 — Internal Productivity (Months 4–6)

Scope: EMAIL (Stalwart-based, Missive-style UX, CaMeL dual-LLM anti-injection); PROJ (Linear-style three primitives Issue/Cycle/Project, sync-engine, Genie cycle-review drafts); KB (ask-this-page Q&A, GraphRAG navigation); OBS full dashboards; TIME (Genie auto-categorisation); CRM (pipeline with confidence-bands, no scoring of people); AI as first-class subgraph; MCP tool registry as first-class subgraph; HR (full); REW (compensation core); LEARN (Hội đồng Chuyên môn workflow).

Out of P1: ESOP, INV (deferred to P2). DOC, PORTAL deferred to P3/P4. Still single-tenant. CUO emergent-role skills (CAIO/CXO/CRO/CSO-Sus) deferred to P2.

P1 → P2 exit gate: EMAIL fully replaced Gmail for ≥21 days; PROJ replaced prior tracker for ≥21 days; CRM has ≥5 active client records, ≥10 deals, Genie next-action acceptance ≥40%; Genie acceptance rate ≥40% on 7-day rolling window for 14 days; auto-pause behaviour either resolved or quarantined; subgraphs independently deployable; first full payroll cycle issued through REW; first promotion review through Hội đồng workflow.

P2 — Operations (Months 7–9)

Scope: HR (employee records, 1:1 templates, onboarding), REW (payslip narrative explainer read-only AI, anomaly surface during close cycle), OKR (quarterly cascade, alignment heatmap), LEARN (career path, peer-review summariser outcomes only), INV (issuance, AR aging, dunning drafts, Stripe + VNPay), RES (capacity vs forecast, staffing simulator), ESOP (grant tracking, vesting, SP put-option simulator). Decree 13 full-regime graduation. CUO emergent-role skills behind feature flags.

P2 → P3 exit gate: payroll cycle close in REW for ≥2 consecutive cycles with zero anomalies escaping; OKR cycle close for ≥1 quarter; all 10+ employees with HR record + signed contract + active LEARN entry; Decree 13 full-regime sign-off by Founder/DPO; ISO 27001 gap-list ≥85% complete; CUO acceptance rate ≥40% across 7-day rolling window; auto-pause triggered and recovered ≥1×.

P3 — SaaS Readiness (Months 10–12)

Scope: full multi-tenancy with residency partitioning (vn-shard, sg-shard, eu-shard); per-tenant theme overrides, DPIA artifacts, CUO persona-version isolation; tenant lifecycle (provision, suspend, archive, delete, export); billing (Stripe non-VN, VNPay VN, metered AI usage); DOC module with eIDAS QTSP signing; GDPR posture on eu-shard; ISO 27001 Stage 1 + Stage 2 audit; SOC 2 Type I report published; Singapore HoldCo flip executed.

P3 → P4 exit gate: first external client (a Vietnamese SME, ideally) provisioned, runs ≥30 days, NPS ≥8; multi-tenant load test passes (100 tenants × 10 users × 1,000 BRAIN ops/day with all SLOs green); ISO 27001 Stage 2 certificate issued; HoldCo flip closed; CUO has produced ≥1 quarterly state-of-business report unaided that the founder accepts with no major edits.

P4 — Client-Facing (Months 13–24)

Scope: DOC + PORTAL + TEN (Tenancy & Billing external) live; first external paying tenant onboarded; multi-tenant external GA opens; T3+ regulated commercial path optional (TX-RAMP, StateRAMP Cat 2, FedRAMP 20x Moderate via no-sponsor route if US sub exists).

How to use this memory

Use this when planning a quarter, scoping a new module, or evaluating whether a request fits the current phase. Anything not on the current-phase scope list is a deferred item, not a "yes but later this phase." Phase exits are signed by the founder; the audit-log entry is the gate artefact.
