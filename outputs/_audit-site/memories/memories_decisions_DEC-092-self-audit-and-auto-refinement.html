DEC-092 — Self-audit + auto-refinement at skill level (registry v0.2.0)

Every Tier-2 CyberOS skill carries `INVARIANTS.md` declaring runtime truths it enforces about its own behaviour. Breaches emit a `refinement_proposal` envelope (new `output_kind`), pause the LangGraph pipeline at a checkpoint, surface as a Question primitive for human review, and resume from the checkpoint after APPROVE / REVISE / REJECT.

What landed in frontmatter

self_audit:
  invariants_ref: ./INVARIANTS.md
  check_at:
    - on_node_boundary          # every LangGraph step
    - on_audit_row_count: 25    # mirrors AGENTS.md §8 consolidation cadence
    - on_completion
  anomaly_signals:
    confidence_low_streak:     {threshold: 3, window: 10}
    user_correction_streak:    {threshold: 2, window: 5}
    denylist_near_miss_streak: {threshold: 2, window: 20}
    scope_rejection_streak:    {threshold: 1, window: 1}
    # plus skill-specific signals (e.g., fr-audit's deterministic_drift)
  on_breach:
    emit: refinement_proposal
    pause_pipeline: true
    resume_token_field: refinement_run_id

Plus new audit row kind `op:"self_refinement_proposal"` and new `produces.output_kind` value `refinement_proposal`. Both extend existing enums additively — no breaking change.

Anatomy of a refinement_proposal

kind: refinement_proposal
skill_id: cuo/cpo/fr-create
skill_version: 0.2.0
trigger: "INV-003 breach: PRD digest has coverage 0.81 (<0.99) without intentional_summary flag"
observation: |
  Read 4 chunks; processed 81% of source lines.
proposed_amendments:
  - tier: 1
    target_doc: cuo/cpo/fr-create/SKILL.md
    section: §"PLAN phase" step 1
    diff: |
      +  Paginate PRD reads sequentially. Track high-water mark.
      +  Verify coverage ≥0.99 OR set intentional_summary: true.
minimum_viable: "Add coverage check at end of PLAN phase."
pipeline_paused: true
resume_token: refinement_run_id_abc123

Relationship to AGENTS.md §0.4

This DEC promotes the §0.4 standing rule (protocol-level: every memory issue triggers a refinement proposal in the same response) to **skill-level**. Each skill now does §0.4-equivalent self-checking at runtime:

| AGENTS.md §0.4 | Skill-level (this DEC) |

| --- | --- |

| Surfaces during **memory operations** | Surfaces during **skill execution** |

| Refines **AGENTS.md** itself | Refines **the skill's SKILL.md / RUBRIC.md / INVARIANTS.md** |

| Recorded in `memories/refinements/REF-NNN-...` | Recorded in `genie.action_log` as `self_refinement_proposal` rows + a follow-up `memories/refinements/` BRAIN entry on adoption |

| Tier 1/2/3 priority + minimum-viable amendment | Same triage; `proposed_amendments[].tier` carries it |

The two layers compose. Same pattern, different surface. AGENTS.md itself doesn't change; this DEC just ports the pattern outward.

Auto-refinement → manual fine-tune escalation

Auto-refinement is for mechanical drift the skill can self-detect and self-propose a fix for. When a skill emits ≥2 refinement_proposals on the same theme within one batch (`self_audit_refinement_proposal_count_above` threshold), the runtime escalates to the manual fine-tune flow (DEC-093). Reason: the auto-refinement loop has flagged a pattern that needs human investigation, not a one-line patch.

How to use this memory

When designing a new CyberOS skill:

1. Author 3-8 invariants in `INVARIANTS.md` covering the skill's highest-value contracts (e.g., fr-audit's INV-001 verdict determinism — the auditor's reproducibility is its core promise).

2. Each invariant has ID + Statement + Check + Severity + Refinement template. The Refinement template is what the runtime emits on breach — keep it actionable (cite the section, propose the diff).

3. `confidence_low_streak` is the most universally-useful anomaly signal — start there if unsure.

4. `pause_pipeline: true` is the default for sev-0 invariants. For advisory invariants (warning-severity), use `pause_pipeline: false` so the skill keeps flowing while logging the concern.
