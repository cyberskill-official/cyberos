Standing rule: ALWAYS suggest memory-protocol refinements when issues surface

Mirror of styx/.cyberos-memory/memories/preferences/feedback-always-suggest-memory-improvements.md.

**Rule.** Whenever a memory issue surfaces — shallow ingestion, missed source content, retrieval miss, duplicate where unique was needed, conflict that resolution-rules don't cleanly handle, the user having to repeat instructions, drift between BRAIN and source, recall of stale facts, denylist false-negative or false-positive, the agent needing to "guess" what's authoritative — the agent MUST proactively propose a refinement to AGENTS.md.

**Why.** AGENTS.md is the design spec for **CyberOS Layer 1 BRAIN**. Every issue is a real-world test case. Surfacing each one as a refinement compounds protocol coverage over time. Silence on a memory issue = lost design feedback for CyberOS.

**Now codified in protocol** at §0.4 (DEC-076). This memory is the human-readable mirror; the protocol is the enforceable contract.

How to use this memory

Load every session-start. Apply on every memory-system interaction.
