REF-011 — Manifest cleanup: schema_version + compatible_runtimes removed (AGENTS.md §6, §4.3, §13.0)

What changed

Removed two manifest fields and replaced the version-comparison forward-compat with field-presence detection. See DEC-086 for the full decision.

What triggered the refinement

Stephen asked "is `compatible_runtimes` and `schema_version` in manifest.json (section 6) necessary?" Analysis showed:

  • `compatible_runtimes`: 1 reference (the example), 0 logic uses → vestigial.
  • `schema_version`: 3 references (§4.3 forward-compat, §13.0 state classifier, the example) but discrete-version-number model is incompatible with day-by-day protocol evolution.

    • Both were removed; field-presence detection achieves the same forward-compat protection without inline version markers. Aligns with the no-inline-versioning rule established earlier the same day.

    Decision record

    DEC-086 in `memories/decisions/DEC-086-remove-schema-version-and-compatible-runtimes.md`.

    How to use this memory

    Load on session-start. When future protocol changes are proposed, prefer field-presence detection over discrete version numbers; prefer permissive-with-surfacing over strict-rejection where the user-visible failure mode is "agent too old."

  • 2026-05-04 — Adopted in the same session DEC-086 was proposed.
