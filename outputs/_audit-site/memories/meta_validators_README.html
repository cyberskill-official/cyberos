Pluggable Validators

Aspect 12.1 of the Layer-1 improvement catalog.

Drop additional validator checks here. Each file is a Python module exporting:

def check(memory: dict, manifest: dict) -> list[Finding]:
    """Return list of {severity, code, message}."""

`cyberos_validate.py` auto-discovers every `*.py` in this directory at run-time

and invokes `check()` on every memory. Project-specific rules without §0.5

protocol amendment.

Example fixtures live in `runtime/tools/tests/validators/` if you want to

include test data.

Severity

  • `CRITICAL` — freezes writes; surfaces in §14 verbose
  • `WARN` — surfaces in §14 verbose
  • `INFO` — logged only

    • Example check

    # meta/validators/check-org-tag-limit.py
def check(memory, manifest):
    tags = memory.get("tags", [])
    if len(tags) > 10:
        return [{"severity": "WARN", "code": "org-tag-limit",
                 "message": "CyberSkill convention: max 10 tags per memory"}]
    return []

    Conventions

  • File name: lowercase-kebab, `check-<rule>.py`
  • One check function per file (single responsibility)
  • Determinism required (same inputs → same outputs)
  • No I/O outside reading the memory dict + manifest dict
