REF-035 — §0.5 rollback flow validated by controlled round-trip test

Real-world trigger

Post-cascade observation: §0.5's rollback path (`op:"protocol_rollback"`, the `meta/protocol-history/` archive flow) was added during Bundle B but never exercised end-to-end. Per the operational principle "rollback is what makes upgrade safe", an unexercised rollback is a phantom safety net — the kind of feature that fails silently until disaster strikes.

Refinement performed

Not a protocol amendment — a **validation exercise**. Ran a controlled round-trip:

1. Saved current AGENTS.md verbatim to `meta/protocol-history/`.

2. Applied a benign HTML-comment test marker.

3. Ran `op:"protocol_upgrade"` to the test state.

4. Ran `op:"protocol_rollback"` back to the original.

5. Verified byte-identical restoration, manifest pin reset, chain LINK intact.

All checks passed. DEC-098 records the validation; this REF records the methodology and operational implication.

Why this is the right fit

Validation rather than amendment because the §0.5 mechanism itself proved correct. The exercise surfaced one operational nuance worth capturing: **the canonical sequence is "archive BEFORE edit"** so the resulting protocol-history file is verbatim and supports rollback. In-turn cascade edits (where Bundle B/A/C/D's pre-states became markers rather than verbatim copies) are an exception — acceptable for the originating session but they limit later rollback granularity for that lineage.

Operational implication

Bundle B/A/C/D's pre-state archives are markers, not verbatim; to rollback to a pre-cascade state would require reconstructing AGENTS.md from git history, not from `meta/protocol-history/`. From this point forward, every protocol_upgrade flow that follows the canonical archive-before-edit sequence creates a verbatim archive that fully supports rollback.

How to use this memory

When designing future protocol-evolution work, prefer one-bundle-per-chat-turn so the canonical archive-before-edit sequence is honoured. When chaining multiple bundles in one turn (like 2026-05-06's cascade), document explicitly that the resulting protocol-history files are markers; users should expect to use git to reconstruct verbatim states for those lineages.
