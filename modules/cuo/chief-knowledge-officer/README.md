# `chief-knowledge-officer` — Chief Knowledge Officer

> Per `../../docs/The C-Suite Reference.md` §5.7 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Knowledge Officer.
- **Persona slug:** `chief-knowledge-officer`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: — · Series A: — · Scale-up: — · Growth: optional · Enterprise: rare except consulting (HIGH ROI for CyberSkill) (per §3 matrix).
- **One-sentence scope:** Rare except in consulting; manages organisational learning assets. **Consulting-firm-specific HIGH ROI** per §7 — institutional IP is the moat.

## §2-§4  Inputs
See C-Suite Reference §5.7. Expand in next session.

## §5  Outputs
**Strategic:** knowledge-taxonomy; IP catalog; reuse-asset portfolio. **Operational:** asset codification cycles; KB freshness audits. **Communication:** internal KB stewardship; client-facing IP portal. **Team:** Knowledge Managers + KB Editors + IP Coders.

## §6  Cadence
Per role profile in §5.7.

## §7  KPIs
| Asset reuse rate | trending up | KB analytics |
| KB freshness % | > 80 % within review-cycle | KB |
| IP-catalog growth | per quarter | IP catalog |
| Time-to-find-asset for consultants | declining | KB analytics |

## §8  Audit criteria
- **Quantitative:** moves KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** KB neglect (write-only); over-IP-protection (consultants can't find); asset rot.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.

## §9  Tools & stack
Notion / Confluence / Slite (KB); Glean / Coveo (search); custom IP portals; Linear / Asana (knowledge sprint mgmt).
