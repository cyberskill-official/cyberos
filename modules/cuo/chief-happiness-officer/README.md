# `chief-happiness-officer` — Chief Happiness Officer

> Per `../../docs/The C-Suite Reference.md` §5.5 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Happiness Officer.
- **Persona slug:** `chief-happiness-officer`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: cultural choice · Series A onward: cultural choice / optional novelty · Enterprise: rare (per §3 matrix).
- **One-sentence scope:** Novelty/branding role; substance overlaps with CHRO. Useful as a culture signal, not a budget center.

## §2-§4  Inputs (information / stakeholder / resource)
See C-Suite Reference §5.5 for the full input lists. Expand in next session.

## §5  Outputs
**Strategic:** culture-rituals framework; happiness-data instrumentation. **Operational:** culture-events; perks budget. **Communication:** culture-keynote at all-hands. **Team:** typically 1-3 people.

## §6  Cadence
Daily / weekly / monthly / quarterly / annual rhythms per role profile. See §5.5.

## §7  KPIs
| Happiness score (survey-driven) | trending up | Officevibe / TINYpulse |
| Culture-events participation | per target | event tracker |
| Glassdoor culture rating | > 4.0 | Glassdoor |

## §8  Audit criteria
- **Quantitative:** moves the persona's 3-5 KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** treating happiness as a perks budget instead of a culture system; metric-gaming on happy-day surveys.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.

## §9  Tools & stack
Officevibe / TINYpulse / Culture Amp (happiness survey); Donut (random-coffee); Glassdoor.
