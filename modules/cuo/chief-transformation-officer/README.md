# `chief-transformation-officer` — Chief Transformation Officer

> Per `../../../modules/cuo/docs/module.md` §5.7 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Transformation Officer.
- **Persona slug:** `chief-transformation-officer`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Project-based; tied to a specific transformation (M&A, restructuring, modernisation) (per §3 matrix).
- **One-sentence scope:** Cross-functional change leadership for specific programs (M&A integration, restructuring).

## §2-§4  Inputs
See C-Suite Reference §5.7. Expand in next session.

## §5  Outputs
**Strategic:** transformation strategy + roadmap. **Operational:** program governance; cross-functional alignment. **Communication:** transformation comms; stakeholder mgmt. **Team:** Transformation PMO.

## §6  Cadence
Per role profile in §5.7.

## §7  KPIs
| Transformation milestone attainment | per program plan | PMO |
| Stakeholder NPS | improving | survey |
| Cross-functional friction score | trending down | program survey |

## §8  Audit criteria
- **Quantitative:** moves KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** transformation-fatigue; scope creep; change-management failure.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.
