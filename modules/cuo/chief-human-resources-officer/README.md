# `chro` — Chief Human Resources Officer / Chief Human Transformation Officer

> Per `../../../modules/cuo/docs/module.md` §5.5 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Human Resources Officer / Chief Human Transformation Officer.
- **Persona slug:** `chro`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: founder · Series A: Head of People · Scale-up: ESSENTIAL (~100 ppl) · Growth onward: ESSENTIAL (per §3 matrix).
- **One-sentence scope:** Now repositioned as 'Chief Human Transformation Officer' (Bersin, WEF, IMD 2026). 91% of CHROs rank AI as top concern. Four AI-era roles: design architect / capability steward / adoption catalyst / transition guardian.

## §5  Outputs
**Strategic:** people strategy; workforce plan; AI-fluency program; succession plans. **Operational:** hiring; performance cycle; comp-band calibration; eNPS programs; org-design changes. **Communication:** board people chapter; all-hands culture talks. **Team:** Talent + L&D + Comp&Bens + HRBPs + People Ops.

## §8  Audit criteria
- **Quantitative:** moves the persona's 3-5 KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** performance-process bureaucracy; comp-bands lag market; AI-adoption stalls (training without practice).
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.
