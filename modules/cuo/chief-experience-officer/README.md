# `cxo` — Chief Experience Officer

> Per `../../docs/The C-Suite Reference.md` §5.4 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Experience Officer.
- **Persona slug:** `cxo`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: — · Series A: — · Scale-up: optional · Growth: common · Enterprise: common (per §3 matrix).
- **One-sentence scope:** End-to-end experience across product + service. Overlaps CCO-Customer / CMO.

## §2-§4  Inputs (information / stakeholder / resource)
See C-Suite Reference §5.4 for the full input lists. Expand in next session.

## §5  Outputs
**Strategic:** experience strategy across all touchpoints; journey-mapping. **Operational:** experience metrics dashboard; cross-functional CX initiatives. **Communication:** journey-map readouts to peer-C-suite. **Team:** UX Research + Service Design + CX Analytics.

## §6  Cadence
Daily / weekly / monthly / quarterly / annual rhythms per role profile. See §5.4.

## §7  KPIs
| End-to-end NPS | > 30 | journey survey |
| Cross-touchpoint friction score | trending down | journey analytics |
| Time-to-first-value | per persona target | product analytics |

## §8  Audit criteria
- **Quantitative:** moves the persona's 3-5 KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** becoming 'CCO-Customer 2.0' without role clarity; journey-mapping without action; experience-metric overload.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.

## §9  Tools & stack
Qualtrics / Medallia (XM); Hotjar / FullStory (session replay); ContentSquare / Quantum Metric (digital experience).
