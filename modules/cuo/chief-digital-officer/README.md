# `chief-digital-officer` — Chief Digital Officer

> Per `../../docs/The C-Suite Reference.md` §5.7 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Digital Officer.
- **Persona slug:** `chief-digital-officer`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: — · Series A: — · Scale-up: — · Growth: project-based · Enterprise: common (per §3 matrix).
- **One-sentence scope:** Digital transformation; often time-bounded (3-5 yrs) then sunsets into CIO/CTO. **Acronym collision** with Data, Diversity.

## §2-§4  Inputs
See C-Suite Reference §5.7. Expand in next session.

## §5  Outputs
**Strategic:** digital transformation roadmap; legacy-system retirement plan. **Operational:** transformation program PMO; partner-integrator mgmt. **Communication:** board transformation chapter; change-mgmt. **Team:** Transformation Program Mgrs + Change Mgmt.

## §6  Cadence
Per role profile in §5.7.

## §7  KPIs
| % of revenue from digital channels | trending up | finance attribution |
| Legacy-system retirement on schedule | per roadmap | PMO |
| Digital-NPS | improving | survey |

## §8  Audit criteria
- **Quantitative:** moves KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** transformation-theatre; partner-integrator dependence; sunsets-prematurely or never.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.

## §9  Tools & stack
PMO tools (Smartsheet, MS Project); transformation-monitoring dashboards.
