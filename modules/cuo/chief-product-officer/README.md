# `cpo-product` — Chief Product Officer (Product)

> Per `../../../modules/cuo/README.md` §5.3 + §4 (9-block schema).

## §1  Identity & scope
- **Full disambiguated title:** Chief Product Officer (Product).
- **Persona slug:** `cpo-product`
- **Reports to:** CEO (typically).
- **Stage prevalence:** Seed: CEO/CTO covers · Series A: optional · Scale-up: common · Growth onward: ESSENTIAL (per §3 matrix).
- **One-sentence scope:** Owns the 'why/what/when'; CTO owns the 'how'. Customer-centric strategy, roadmap, pricing, market-fit.

## §2-§4  Inputs (information / stakeholder / resource)
See C-Suite Reference §5.3 for the full input lists. Expand in next session.

## §5  Outputs
**Strategic:** product vision + roadmap; pricing strategy; market-fit thesis. **Operational:** quarterly roadmap commitments; pricing changes; feature-launch sign-offs. **Communication:** board product chapter; all-hands product keynote; customer advisory board. **Team:** VP Product / Product Managers / Product Designers / Product Analytics.

## §6  Cadence
Daily / weekly / monthly / quarterly / annual rhythms per role profile. See §5.3.

## §7  KPIs
| NPS | per industry benchmark | survey |
| Activation rate | per onboarding funnel | product analytics |
| Retention (D7/D30) | per business model | product analytics |
| Feature adoption | per launch target | product analytics |
| Time-to-value | per product type | product analytics |

## §8  Audit criteria
- **Quantitative:** moves the persona's 3-5 KPIs above; forecast within ±10 %.
- **Qualitative rubric:** alignment / coherence / customer-grounding / risk-realism / communicability.
- **Universal failure modes (per §6):** playbook transplant; activity over outcomes; silo-ing; forecast drift without narrative; hero dependence; AI-washing.
- **Role-specific failure modes:** feature factory (shipping without retention); pricing-tier confusion; PM-CTO turf wars on 'what' vs 'how'; not killing failed bets.
- **Commercial baseline reminders (per §8):** don't add a C-title for a process problem; match title to stage; disambiguate acronyms; beware hype-cycle titles; audit outputs not titles.

## §9  Tools & stack
Linear / Jira (roadmap); Productboard / Aha! (vision); Amplitude / Mixpanel (analytics); Pendo / Appcues (in-product); Figma (design); Maze / Dovetail (research).
