# security-strategy@1 — Annual information security strategy

A `security-strategy@1` artefact authors the annual information-security strategy — threat landscape assessment, current posture (vs NIST CSF 2.0 functions), prioritized initiatives, budget envelope, OKRs, board narrative. Per NIST CSF 2.0 + ISO/IEC 27001:2022 + CIS Controls v8 implementation groups.

## Required sections (template.md H2 order)
1. Threat landscape (per Verizon DBIR + industry-specific intel)
2. Current posture vs NIST CSF 2.0 (Govern / Identify / Protect / Detect / Respond / Recover)
3. Maturity self-assessment per CIS IG1/IG2/IG3
4. Prioritized initiatives (per posture-gap, with NIST CSF function mapping)
5. Budget envelope + investment thesis
6. Security OKRs for the year
7. Board narrative (1-page chapter)
8. Risks & open issues

## Citations
- NIST Cybersecurity Framework 2.0
- ISO/IEC 27001:2022
- CIS Controls v8 (IG1 / IG2 / IG3)
- Verizon DBIR (annual data breach investigations report)
- MITRE ATT&CK threat-actor mapping

## KPI
- Posture-gap closure vs initial assessment
- Incident MTTR
- Audit findings count (trending down)
