2026-05-22 — Unified cyberos workflow CLI + API extraction + workflow stability fixes

Unified CLI: Added cyberos workflow subcommand to the main cyberos CLI. Users can now run workflows from any project without needing the separate cyberos-cuo command:

  • cyberos workflow <persona-workflow> [--rework] — drain eligible FRs
  • cyberos workflow list-personas — enumerate personas
  • cyberos workflow list-workflows <slug> — enumerate workflows
  • cyberos workflow dry-run <path> — plan without executing

API extraction: Created cuo/api.py — pure-Python functions (no Click dependency) for all CUO workflow operations. Refactored cuo/cli.py to delegate to the API layer. cyberos-cuo still works for backward compatibility.

Workflow stability from external projects:

  • CYBEROS_ROOT env var — primary discovery mechanism for the cyberos repo root (checked before CWD walk in _find_cyberos_root)
  • _find_backlog_md falls back to repo root when output_dir is outside the repo tree (fixes silent applier failure)
  • execute_chain injects _cyberos_root into hand_off for applier access
  • select_invoker now warns visibly when falling back to MockInvoker
