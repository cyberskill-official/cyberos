PLUGIN

P1 · Productivity (Distribution) Specced 2026-05-19 · 8 FRs at 10/10 · runtime build-out next session Owner: CTO seat (Stephen)

The distribution surface. Turns shipped CyberOS modules into installable plugins for any agentic IDE — Claude Code, Cursor, Anthropic Cowork, OpenAI Codex CLI — without the user needing to know the federation exists.

PLUGIN sits one layer above the MCP gateway. Where MCP owns the protocol (spec compliance, OAuth handshake, tasks primitive, elicitation), PLUGIN owns packaging + multi-runtime emission + marketplace publish. A single canonical manifest at modules/plugin/manifests/<id>@<version>.plugin.json bundles into 4 target-native artefacts (`.plugin`, `.mcp.json`, Customize zip, SKILL.md folder), each signed via Sigstore Rekor and pushed to plugins.cyberskill.world with public-tier mirror to agentskills.io. Strategy alignment: §4 Level 1 (OSS distribution) + Level 3 (marketplace with 70/30 revenue share + vetted-by-CyberSkill JWT badge) + Level 4 (vertical packs ship as cyberos-vn / cyberos-sg / etc. plugins) + Level 5 (enterprise white-label private marketplace origins). The planned runtime lives at cyberos/services/plugin-host/.

Why a separate module — not "just an MCP gateway feature"

The MCP gateway owns the protocol. PLUGIN owns the distribution + packaging surface. Keeping them separate matches CyberOS's "agentic-native + open-standard + audit-chained + regionally-localized" positioning without overloading the gateway with marketing-surface mechanics.

Auth + audit — every action audit-chained, every token short-lived

OAuth-PKCE (FR-PLUGIN-005)

  • ✓ Audience-bound JWT RS256 (aud: "plugin:<id>") — blocks cross-plugin token reuse
  • ✓ 1-hour access tokens; 24-hour rotating opaque refresh
  • ✓ OS-keychain storage (macOS Keychain · Windows Credential Manager · Linux Secret Service)
  • ✓ 7 locked scopes: cyberos:{cuo,memory,skill}:{list,read,write,execute,route,invoke}
  • ✓ Revocation propagates within 60 seconds via cache TTL

memory audit (FR-PLUGIN-006)

  • ✓ 6 audit kinds: plugin.installed / updated / uninstalled / invoked / auth_refreshed / scope_denied
  • ✓ Audit-then-respond ordering — no lost rows on crash
  • ✓ Durable Postgres outbox · 24h exponential backoff retry
  • ✓ SHA-256 idempotency key dedups retries server-side
  • ✓ Body strictly scrubbed — no tool input / output leak

Strategy fit — the substrate for every distribution play

Without PLUGIN, CyberOS is only installable via git clone or direct download — viable for early adopters, fatal for ecosystem distribution. With PLUGIN, the same canonical CyberOS appears as a one-click install across every major agentic IDE.

Strategy levelWhat unlocksThis module ships
Level 1 — OSS distributionDevelopers clone, install, use CyberOS without on-boarding from CyberSkillPublic visibility · agentskills.io mirror · Apache-2.0 default
